# SSG Prerender для SEO

**Дата**: 2026-04-01
**Цель**: При `vite build` генерировать HTML с контентом, чтобы краулеры видели страницу без JS.

## Контекст

Сейчас SPA — `<div id="app"></div>` пустой, всё рендерится JavaScript. Краулеры видят пустую страницу. SEO-шник просит отдавать контент без JS.

## Подход

Кастомный prerender-скрипт на Node.js:
1. `vite build` собирает проект как обычно
2. Скрипт рендерит Vue-приложение в HTML-строку через `@vue/server-renderer`
3. Вставляет результат в `dist/index.html` вместо пустого `<div id="app"></div>`
4. Vue на клиенте монтируется поверх существующего DOM и оживляет интерактивные элементы

## Что рендерится в статике

| Элемент | В статике | Через JS |
|---------|-----------|----------|
| Лого "Ведомости" | ✅ | — |
| Заголовок + подзаголовок | ✅ | — |
| Фоновая картинка (медведь) | ✅ | — |
| Форма email (визуально) | ✅ | Отправка |
| Футер (иконки, текст, ссылки) | ✅ | — |
| Таймер обратного отсчёта | ❌ | ✅ |
| Карточки RSS | ❌ | ✅ |
| Cookie-баннер | ❌ | ✅ |
| Смена темы (орёл/жук) | ❌ | ✅ |

## Архитектура

### Файлы

```
scripts/
  prerender.ts        — скрипт пререндера (~50-70 строк)
src/
  App.vue             — без изменений
  AppSSG.vue          — обёртка для SSG: тема=медведь, без таймера/карточек/cookie
  main.ts             — без изменений (клиентский entry)
  entry-server.ts     — серверный entry для @vue/server-renderer
package.json          — новый скрипт "build": "vue-tsc -b && vite build && node scripts/prerender.js"
```

### AppSSG.vue

Минимальная версия App.vue для SSG:
- Тема зафиксирована на медведе (единственный фон в статике)
- Без `onMounted` (нет DOM на сервере)
- Без таймера, карточек, cookie-баннера
- HeroSection получает пустой массив `cards` и `submitted=false`
- Футер рендерится полностью (статический контент)

### entry-server.ts

```ts
import { createSSRApp } from 'vue'
import { renderToString } from 'vue/server-renderer'
import AppSSG from './AppSSG.vue'

export async function render(): Promise<string> {
  const app = createSSRApp(AppSSG)
  return await renderToString(app)
}
```

### scripts/prerender.ts

1. Импортирует `entry-server.ts` (через `vite.ssrLoadModule` или прямой import после SSR-билда)
2. Вызывает `render()` → получает HTML-строку
3. Читает `dist/index.html`
4. Заменяет `<div id="app"></div>` на `<div id="app">{html}</div>`
5. Записывает обратно

### Процесс билда

```
vite build                    → dist/index.html (SPA, пустой div)
vite build --ssr entry-server → dist-ssr/entry-server.js
node scripts/prerender.js     → dist/index.html (с HTML-контентом)
```

### Клиентская гидрация

`main.ts` остаётся без изменений. Vue 3 `createApp().mount('#app')` автоматически переиспользует существующий DOM если он совпадает со структурой компонентов. 

Несовпадение допустимо: клиент рендерит полную версию (с таймером, карточками), SSG рендерит минимальную. Vue заменит содержимое `#app` на полную версию при монтировании — это стандартное поведение SPA, просто пользователь увидит контент раньше.

## Важные ограничения

- **Нет гидрации в строгом смысле** — Vue не "оживляет" серверный HTML, а заменяет его своим. Это нормально для данного случая: цель — показать контент краулеру, не оптимизировать TTI.
- **Статика устаревает** — HTML обновляется только при деплое. Для этого проекта это приемлемо (контент меняется редко).
- **Одна тема в статике** — медведь. При загрузке JS может смениться на орла/жука (будет мгновенный flash, но это допустимо).

## Что НЕ меняется

- Вся текущая функциональность
- Структура компонентов
- Деплой на Vercel (по-прежнему `vite build`, просто добавляется шаг пререндера)
- `main.ts`, `App.vue`, все компоненты
